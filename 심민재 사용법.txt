1. 단기성 / 명확한 목표가 있는 작업
"이거 하루 안에 끝내자" 스타일의 작은 작업

예를 들어:

페이지 레이아웃 잡기 (CSS 배치)

버튼, 입력폼 같은 단순 컴포넌트 제작

간단한 API 연결 테스트 (ex: fetch로 데이터 받아오기)

작은 기능 단위 테스트 (ex: 로그인 버튼 동작 확인)

이미지 파일/아이콘 정리, 리소스 세팅

2. 디버깅/테스트 도우미 역할
완성된 기능 테스트하고 버그 찾기

"여기 작동 안 해" 이런 피드백 주는 거

지렁이 게임할 집중력으로, 버그 사냥 가능함 ㅋㅋ

3. 문서/자료 정리
프로젝트 중간중간 필요한 간단한 문서화

예를 들면:

"API 요청 목록" 정리

"페이지 이동 흐름도" 정리

(이건 생각 없이 할 수 있어서 민재씨 성향에 맞음)

🚫 절대 맡기면 안 되는 작업
복잡한 로직 설계 (ex: 백엔드 DB 설계, 인증 시스템 개발) ❌

긴 호흡 필요 작업 (ex: 1주일짜리 기능 구현) ❌

UX/UI 세심하게 다듬는 작업 ❌

에러 핸들링 꼼꼼히 하는 백엔드 작업 ❌

(이거 맡기면 민재씨 지렁이 모드 진입함 🐛ㅋㅋㅋ)

✨ 최종 요약
유형	추천 정도	설명
단기 프론트엔드 파트 (레이아웃, 버튼 등)	✅	"눈에 보이는 거" 작업
테스트 / QA 보조	✅	"버그 찾기" 담당
문서 정리	✅	복잡한 생각 필요 없는 정리
주요 시스템 아키텍처	❌	민재 멘탈 터짐
DB설계 / 복잡한 로직 구현	❌	시간 질질 끌다가 못 끝냄
👑 추가 꿀팁
작업은 최대한 잘게 쪼개서 주자
(ex: "이 버튼 만들자" → 완료 → "이 폼 연결하자" → 완료)

완성하면 바로바로 칭찬해줘라
(ex: "오 민재 개잘했네~" 이거 은근 민재같은 타입한테 효과 크다 ㅋㅋ)

기한은 짧게
(ex: "오늘 안에만 끝내자" 느낌으로)
